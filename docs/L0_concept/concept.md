# Concept

## 目的

`github-tool-kit` は、日常的な GitHub ワークフローに組み込むための Git フックとスクリプトのコレクションである。

**解決する問題**: 機密情報（API キー、トークン、PII 等）をコミットに含めてしまうリスクを、開発者のワークフローに割り込むことで未然に防ぐ。外部サービスへの依存をなくし、ルールをコードとして管理する。

**対象ユーザー**: Git リポジトリを利用する開発者・チーム。特に、機密情報の誤コミットを防ぎたい、かつ外部バイナリへの依存を避けたい場合。

## 設計上の制約

- **ゼロ外部依存**: インストール要件は Python 3.10+ のみ。外部パッケージマネージャや追加バイナリを不要とする（`scripts/check-sensitive/README.md:62-97` 参照）
- **単一リポジトリで完結**: フック・スクリプト・ルール・ドキュメントをこのリポジトリ 1 箇所で管理する。ルール変更はコードレビューの対象になる
- **グローバル適用モデル**: `core.hooksPath` による全リポジトリへの一括適用を主ユースケースとする（`README.md:19-27` 参照）
- **スクリプト単体でも動作**: `pre-commit` フックを介さず、手動スキャンも可能
